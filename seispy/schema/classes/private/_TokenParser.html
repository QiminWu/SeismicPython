seispy.schema._TokenParser – _TokenParser class¶

class seispy.schema._TokenParser(input_source)[source]¶

A parser to parse schema definitions.

This class provides functionality to parse schema definitions into cohesive units. Parsable units are blocks, fields, keys, and values.

Parameters:input_source (str) – path to file to be parsed OR a str containing schema definition

get_block()[source]¶

Buffer a block of data.

A block of data is the largest cohesive unit of data _TokenParser can parse. A block is an entire Attribute definition, entire Relation definition or a schema definition header. This method will both return the parsed block and update the _block attribute.

Returns:next block of data
Return type:str
get_field()[source]¶

Buffer a field of data.

A field of data is one key-value pair. This method will both return the parsed field, and update the _field attribute.

Returns:next field in block if one exists, otherwise None
Return type:str`or :data:`None
get_key()[source]¶

Return field key.

A key is, along with a value, one of the two smallest cohesive data units. Examples of keys in an attribute definition are Format, Detail and Null.

Returns:key in buffered field
Return type:str or None
get_value()[source]¶
