seispy.schema.Schema – Schema class¶

class seispy.schema.Schema(path=None, schema='css3.1')[source]¶

Container class providing storage for schema definitions.

This class also provides functionality to write Sphinx documentation pages for schema definitions.

Parameters:
  • path (str) – path to schema definition
  • schema (str) – schema name

If path is provided, Schema instance will be initialized from definition found in path. If path is not provided and schema is, Schema instance will be initialized from installed package definition of schema.

_parse_attribute()[source]¶

Parse contents of an attribute definition.

Assuming the block of data currently stored by _token_parser is an attribute definition, this method will parse the contents of the block and will add to attributes a new _Attribute instance representing the block contents.

_parse_body()[source]¶

Parse body of schema definition.

The body is considered all attribute and relation definitions. This method uses a _TokenParser to read coherent blocks of data and calls _parse_attribute() or _parse_relation(), depending on the result returned by _TokenParser.get_key().

_parse_header()[source]¶

Parse header contents of schema definition.

Schema definitions begin with a header to provide the name, description and details about the schema. This method will parse the contents of the schema definition header, with the ability to handle Schema, Description, Detail, and Timedate keywords.

_parse_relation()[source]¶

Parse contents of a relation definition.

Assuming the block of data currently stored by _token_parser is an relation definition, this method will parse the contents of the block and will add to relations a new _Relation instance representing the block contents.

_write_schema_elements(subdir, elements=None)[source]¶

Generate Sphinx documentation pages for schema element definitions.

This method is a wrapper to iterate over element (attributes/relations) list, calling each instance’s sphinx_documentation_block() method.

_write_schema_element_index(subdir, key='all', elements=None, indices=None)[source]¶

Generate index page for indexed schema elements.

A schema element is an attribute or a relation. This method will generate an index page for schema elements grouped by first letter. (Eg. an index page for all attributes falling under the index ‘A’ - algorithm, amp, anet...)

Parameters:
  • subdir (str) – the name of the sub-directory within the target output directory to write grouped schema element index page to
  • key (str) – the key (character) to generate index page for
  • elements (str) – the schema element (attributes, relations) to generate index page for
  • indices (list) – a list of indices with a non-zero set of indices required to generate the index page for key all
_write_schema_index(subdir, attr_indices=None, rel_indices=None)[source]¶

Generate index page for schema instance.

Index page consists of links to alphabetized indices of both schema attribute definitions and schema relation definitions, as well as details on the schema if available.

Parameters:
  • subdir (str) – The name of the sub-directory within the target output directory to write schema index page to.
  • attr_indices (list) – a list of alphabetic indices that have a non-zero set of attributes
  • rel_indices (list) – a list of alphabetic indices that have a non-zero set of relations

New in version 0.0alpha.
