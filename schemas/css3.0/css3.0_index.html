Detail¶

Modifications from original CSS documentation:
  1. units of calib vary according to the instrument, with wfdisc.segtype and instrument.rsptype indicating both sensor type and units
  2. Null values corrected for certain attributes.
  3. offdate added to primary keys for tables in which it occurs.
  4. endtime added to primary keys for tables in which it occurs.
  5. time made first primary key in origin for sorting.
  6. arid and orid added to foreign keys in assoc.
  7. made range values expression for automated testing
  8. added wfedit relation 12/3/93
  9. changed the primary key in sitechan to chanid, and added chanid as a foreign key in sensor to force joins of sitechan to go through sensor table.
  10. changed primary keys in moment and centryd table to orid.
  11. added calibration and stage tables 1/31/94
  12. changed primary keys in stamag to arid, magtype, sta, orid
  13. changed primary key in site to sta (no ondate, offdate)
  14. changed null values for origerr’s covariant matrix
  15. changed definition of ndef for origins included from other catalogs
  16. added beam, fkgrid and stgrid tables to accomodate array processing 12/15/94
  17. added wftar table to accomodate tar tape waveform archiving 1/9/95
  18. changed all NONULL null values to reasonable values
  19. added wfrms table
  20. added wfmeas table for holding generic waveform measurements
  21. segment origin and stassoc etype field into two fields, etype and review, so that analyst review status can be kept in origin table
  22. add snetsta, anetsta, schanloc and achanaux tables to translate between foreign volumes of SEED or autoDRM into local databases.
  23. add specdisc table and associated attributes to support spectral estimation processing
  24. add rsprm to specdisc table
  25. added tables dmcseed and dmcwf to database to support all the DMC requirements for building DMC seed volumes. Changed default value of fileno to -1.
  26. changed format of chksum from Integer %15ld to Real %12.0f to make sure table is written properly. Previous format would wrap to negative numbers in some cases which would corrupt the database.
  27. added fields calib, calper, samprate, timever to table dmcwf and field totbytes to table dmcseed.
  28. changed primary keys of stamag from arid magtype sta and orid to magid magtype sta and orid.
  29. changed fields dmcseedfile to dfile and jdate to yearday in tables dmcwf and dmcseed.
  30. added focal mechanism calculation related tables and emodel table.
  31. extensive changes to wfrms table to make compatible with orbwfrms.
  32. made instrument table alternate key all fields except inid and lddate.
  33. introduced new calibration table, with new fields samprate, segtype, dlsta, dlchan, and lead.
  34. changed primary keys of origin to include ndef and nass.
  35. made new table mt to hold moment tensor solutions as reported by USGS GeoJSON data feed.
